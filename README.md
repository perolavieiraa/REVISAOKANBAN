O projeto tem como objetivo desenvolver uma aplicação web de gerenciamento de tarefas para uma indústria alimentícia que utiliza o método kanban em formato simples. A aplicação foi construída com HTML, CSS, JavaScript e MySQL, permitindo o cadastro, visualização e atualização de tarefas de forma prática e integrada entre os setores da empresa. A aplicação conta com três telas principais. A primeira é a de cadastro de usuários, onde o sistema solicita nome e e-mail, realiza uma validação do endereço e exibe a mensagem “Cadastro concluído com sucesso” ao final do processo. Em seguida, há a tela de cadastro de tarefas, que permite registrar novas tarefas vinculadas a usuários já cadastrados. Todos os campos são obrigatórios ter descrição, setor, prioridade (baixa, média ou alta) e status inicial “A Fazer” garantindo a consistência das informações. A terceira é a tela de gerenciamento de tarefas, que é a principal do sistema. Nela, as tarefas são exibidas em três colunas, “A Fazer”, “Fazendo” e “Pronto”, conforme o status atual. O usuário pode visualizar os detalhes de cada tarefa, editar informações, excluir registros e alterar o status diretamente na interface. A exclusão solicita confirmação antes da remoção definitiva e a edição redireciona para o formulário com os dados já preenchidos. O banco de dados MySQL foi estruturado com duas tabelas principais: usuarios (id, nome e e-mail) e tarefas (id_tarefa, id_usuario, descrição, setor, prioridade, data de cadastro e status), mantendo um relacionamento entre elas. Todos os campos são obrigatórios para assegurar a integridade dos dados.
